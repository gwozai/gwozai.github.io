第一步：安装 NVM

1. 下载 NVM

下载 NVM 0.40.3 版本的压缩包：

📥 下载 NVM 0.40.3
2. 解压 NVM

创建 .nvm 目录并解压文件：

# 创建 .nvm 目录 mkdir -p ~/.nvm # 解压 nvm-0.40.3.tar.gz 到 .nvm 目录 tar -xzf nvm-0.40.3.tar.gz --strip-components 1 -C ~/.nvm
3. 配置环境变量

根据你的 Shell 类型选择配置：

对于 zsh（~/.zshrc）：

echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc echo '[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"' >> ~/.zshrc echo '[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"' >> ~/.zshrc

对于 bash（~/.bashrc）：

echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.bashrc echo '[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"' >> ~/.bashrc echo '[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"' >> ~/.bashrc
4. 重新加载配置
source ~/.zshrc # 如果使用 zsh # 或 source ~/.bashrc # 如果使用 bash
5. 验证 NVM 安装
nvm --version # 应显示 0.40.3
✅ 成功提示

如果显示版本号 0.40.3，说明 NVM 安装成功！

第二步：安装 Node.js 14

1. 下载 Node.js 14

下载 Node.js 14.21.3 版本：

📥 下载 Node.js 14.21.3
2. 创建目标目录
mkdir -p ~/.nvm/versions/node/v14.21.3
3. 解压 Node.js
tar -xzf node-v14.21.3-darwin-x64.tar.gz --strip-components 1 -C ~/.nvm/versions/node/v14.21.3
4. 使用 Node.js 14
nvm use 14.21.3
5. 验证安装
node --version # 应显示 v14.21.3 npm --version # 应显示对应的 npm 版本
✅ 安装成功

如果两个命令都显示正确的版本号，说明 Node.js 14 安装成功！

第三步：配置和管理

设置默认版本

设置 Node.js 14 为默认版本：

nvm alias default 14.21.3

设置其他版本为默认（如需要）：

nvm alias default 16.20.1
常用 NVM 命令
nvm ls # 列出已安装的版本 nvm use <版本> # 切换版本 nvm current # 显示当前使用的版本 nvm alias default <版本> # 设置默认版本
优化 npm 配置

设置国内镜像源（加速下载）：

npm config set registry https://registry.npmmirror.com

其他 npm 配置命令：

# 查看当前配置 npm config list # 恢复官方源 npm config set registry https://registry.npmjs.org/
目录结构参考
~/.nvm/ ├── nvm.sh # NVM 主脚本 ├── bash_completion # 命令补全脚本 ├── versions/ │ └── node/ │ └── v14.21.3/ # Node.js 安装目录 │ ├── bin/ # 可执行文件目录 │ ├── include/ # 头文件目录 │ ├── lib/ # 库文件目录 │ └── share/ # 共享资源目录

第四步：故障排除

NVM 命令未找到
⚠️ 常见问题

如果终端显示 "command not found: nvm"，请按以下步骤检查：

  • 检查 ~/.nvm 目录是否存在
  • 确认环境变量是否正确配置
  • 重新加载 shell 配置文件：source ~/.zshrc 或 source ~/.bashrc
Node.js 版本未被识别
⚠️ 版本问题

如果 nvm ls 中看不到安装的版本，请检查：

  • 检查版本目录结构是否正确（~/.nvm/versions/node/v14.21.3）
  • 确保文件权限正确（chmod +x 可执行文件）
  • 运行 nvm ls 检查版本列表
npm 不可用
⚠️ npm 问题

如果 npm 命令无法使用，请尝试：

  • 检查 bin 目录下的可执行文件
  • 确保文件具有执行权限（chmod +x）
  • 重新加载 NVM：source ~/.nvm/nvm.sh
注意事项
💡 重要提示
  • 本指南适用于 macOS 系统，其他系统需调整路径和命令
  • 下载链接可能因网络环境而失效，请检查链接有效性
  • 建议定期检查 NVM 和 Node.js 的官方更新
  • 如果遇到权限问题，避免使用 sudo，正确配置用户权限
